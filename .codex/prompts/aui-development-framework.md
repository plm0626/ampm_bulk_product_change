# AUI Development Framework Prompt

When working in this repository:

1. Start with `README.md` and the relevant file in `docs/use-cases/`.
2. Update the use case when behavior changes.
3. Keep the Related Code section accurate.
4. Add or update tests for user-facing and admin-facing behavior.
5. Add issue drafts under `docs/github-issue-drafts/` for known gaps.
6. Keep shared or reusable patterns in documented helpers.
7. Avoid absolute symlinks and machine-specific paths.
8. Run `make check` before committing when the toolchain is available.
